Sleek Designs for Improving Copying to Clipboard Experience

Jun 24, 2019 | 4 min read

The default way (no sleek designs involved)

Since copying is an essential command, there is a default way to do it. Let’s look at the typical user flow:

The default way (featuring my favorite ipsum tool — neilipsum.pw)
  1. User hovers over text and positions the cursor at the beginning (or the end) of the string
  2. User clicks and drags the mouse until the string is highlighted
  3. User opens the context menu with a right- or control-click
  4. User clicks “Copy” (can also be accomplished with a keyboard shortcut)

Unfortunately, this is not the most user-friendly approach. Although it comes for free, it brings along a few little annoyances:

  • First, it can take several seconds to properly position the cursor.
  • Second, once the user starts highlighting the string, they may realize that they didn’t position the cursor properly in the first place.
  • And third, even after they successfully highlight the string, they may accidentally click away and lose the highlight.

The good news is that we can help the user to avoid these annoyances by creating an easier way to copy. While we can’t always predict which bits of text the user will want to select, these examples could be a safe bet:

  • Email addresses
  • Translations
  • Passwords
  • Commands
  • Discount codes

With that being said, let’s look at…

The improved way

A better way to approach this problem is to allow user copy text without highlighting it. We can achieve this by adding a copy-on-click button next to the text, as seen in the following examples from Bitbucket and Google Translate:

Bitbucket

Bitbucket

Clicking on the icon copies the checkout command. When you hover over the icon, a neat tooltip appears and informs you of the upcoming action. However, after you click the icon, some things get unclear:

  • First, there’s no feedback confirming that you copied text to clipboard
  • Second, since there’s no feedback and the text is highlighted, you may assume that clicking the icon only highlighted the text, without actually copying it

Google Translate

Google Translate

Clicking on the icon in the bottom right corner will copy the translation. Just like in Bitbucket, hovering over the icon will display a “Copy” title. Unlike in Bitbucket, however, there’s also a confirmation tooltip.

One drawback to Google’s design is that the icon is positioned too far from the text. If the user doesn’t intentionally explore the icons, they may not even become aware of the feature.

* * *

While adding a button is a good option, it requires the user to learn and follow a new procedure. In addition, there’s not a widely recognized icon for copying to clipboard, which can make it difficult to spot the functionality. Therefore, if you still decide to incorporate this feature into your design, it might be a better idea to use text instead, as seen on Unsplash:

Unsplash

Personally, I am more likely to use a button with text than with some ambiguous icon. Whenever I see an icon, I usually automatically opt-in for the default way to copy, even if it takes longer.

With this in mind, could there still be a better way to copy text? In my opinion, yes.

The Best Way (I’ve seen so far)

A few days ago, I had to reset my password in npm. I needed to copy a temporary password and that’s how it went:

Resetting password for npm

So what’s going on here? Let’s break it down:

  1. I hover over the password to start highlighting and… BOOM! The moment I hover, it highlights “on its own”.
  2. I click to open the context menu and instead see a message notifying me that the string was already copied to clipboard.

This flow is nearly identical to the default way of copying — the only difference is that npm always remains one step ahead. It takes care of all the little annoyances before they can occur:

  • #1 ensures that I never have to worry about highlighting the string
  • #2 frees me from having to select the proper command and prevents me from accidentally clicking away

In addition, I don’t have to follow a new procedure. I keep doing what I usually do and npm takes the responsibility to make the process easier, instead of making me learn how to do it myself.

* * *

While npm offers a very neat method, it obviously would not work everywhere. Not every case is as straightforward and we can’t always predict the exact string the user will want to copy. We also wouldn’t want to frustrate the user by mistakenly getting in their way.

Nevertheless, we should search for elegant solutions whenever we can. The combination of little improvements like these is what truly creates a better experience.
