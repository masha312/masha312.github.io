Step 1: Text

Our first step will be setting up html. This will be simple: we will have a <p> (paragraph tag) with text wrapped inside a <div> (generic container tag).

<div id="container">
  <p>[your text here]</p>
</div>

We will also give an id to our <div> tag. You can name it any way you want‚Äî it doesn‚Äôt have to be ‚Äúcontainer‚Äù. Later, we will be using this id for styling.

Step 2: Positioning

Now, let‚Äôs use the id we assigned to <div> tag to position the text in the center, like this:

There will be two parts: first, we will give our <div> container width, height, and place it in the center of the screen. Second, we will center the contents of the container (in our case, the <p> text) within it.

For the first part, copy and paste this in the CSS section of the editor:

#container {
  position: absolute;
  height: 100%;
  width: 800px;
  left: 0;
  right: 0;
  margin: 0 auto;
}

This is what the result should be:

Orange square represents the <div> container

For the second part, add the following to #container in CSS:

display: flex;
align-items: center;
justify-content: center;

So the end result will be:

#container {
  position: absolute;
  height: 100%;
  width: 800px;
  left: 0;
  right: 0;
  margin: 0 auto;
  display: flex;
  align-items: center;
  justify-content: center;
}

These last 3 flex properties center the <p> text in the middle of our <div> container. This is what it looks like now:

Step 3: Font properties

Now we will use some CSS font properties to make our text look more like a sign. We will apply the following to the <p> tag:

  1. font-size: 80px to make our text bigger.
  2. font-family: Roboto, sans-serif to make it look more modern. (Read more about serif vs. sans serif fonts here).
  3. letter-spacing: 2px to give more room for the neon glow later on.
  4. color: rgba(0, 0, 0, 0.9). This is will keep the color of the text black, while slightly bringing down the opacity to make it less harsh on the eyes.

So, this is the code you should add to the CSS portion of the editor:

p {
  font-size: 80px;
  font-family: Roboto, sans-serif;
  color: rgba(0, 0, 0, 0.9);
  letter-spacing: 2px;
}

As you can see, I didn‚Äôt add an id to my <p> tag and just applied styles to the<p> tag itself. It‚Äôs totally up to you whether you want to add an id to the <p> tag as well (just like we did with the <div> tag) and target it specifically.

Step 5: Shadows

And now we get to play üéâ

There are two parts to creating a neon glow effect: first, we will use the text-shadow property to create several variations of a color glow around the text and second, we will use the animation property to make these variations to transform from one to another.

Step 5 deals with the first part ‚Äî adding the text-shadow property to create a color glow around the text.

text-shadow: [horizontal offset] [vertical offset] [blur] [color];

For example, text-shadow: 20px 10px 7px rgba(0, 0, 0, 0.25); will look like this:

20px to the right, 10px down, 7px blur, and black color with 25% opacity

Another important aspect of the text-shadow property is that it allows to layer several shadows on top of each other. This is the key element to creating a neon effect.

For example, text-shadow: 20px 10px 7px black, -10px 25px 3px red, 0px -50px 20px green will look like this:

3 shadows layered on top of each other

This was lot! Here‚Äôs good news though: to create a color glow, we will only need to concentrate on two values ‚Äî blur and color. Horizontal and vertical offset will always be 0px.

Now, you should pick 2‚Äì3 colors for your neon sign (you can use the Google color picker) and play with the text shadow property. You should come up with 2‚Äì4 color glow variations that we will then use for the animation.

For example, I picked two colors ‚Äî pink (#f50abe) and blue (#0097e8). Here are my variations:

Variation 1

text-shadow: 0px 0px 9px #0097e8, 0px 0px 15px #f50abe, 0px 0px 5px #f50abe;

Variation 2

text-shadow: 0px 0px 9px #f50abe, 0px 0px 15px #0097e8, 0px 0px 5px #f50abe;

Variation 3

text-shadow: 0px 0px 9px #0097e8, 0px 0px 15px #f50abe, 0px 0px 5px #0097e8;

While the difference between these variations may appear negligible, it will become apparent once we add the animation.

Make sure to save your variations somewhere and keep one of them as the text-shadow property. This is what you should have by the end of Step 5:

I set one of the variations as initial neon glow and commented out the other two to save them for the animation

Step 6: Animation

Animation is another CSS property that can take a variety of values. For the sake of this tutorial we will only be using three: name, duration, and iteration count:

animation: [name] [duration] [iteration count]

You can name your animation however you want. This name will be used to bind the animation itself to the text that has the animation property (in our case, the <p> text). In this example, I will name my animation ‚Äúglow‚Äù.

Duration value specifies how long it should take an animation to complete one cycle and iteration count specifies the total number of times the animation should be played. For my animation, I want one cycle to last 10 seconds and I want it to run infinitely.

Therefore, this will be my animation property:

animation: glow 10s infinite;

This is what it looks like in CodePen:

However, we don‚Äôt see any animation yet because we haven‚Äôt created the animation itself. We will do so using @ keyframes rule, which will allow us to change styles of the text at certain times. You can add the @ keyframes rule at the bottom of the CSS section in the editor.

Now it‚Äôs time to use the variations which we came up with in step 5. You should have something similar to this:

@keyframes glow {
  0% {
    text-shadow: 0px 0px 9px #0097e8, 0px 0px 15px #f50abe, 0px 0px 5px #f50abe;
  }   50% {
    text-shadow: 0px 0px 9px #f50abe, 0px 0px 15px #0097e8, 0px 0px 5px #f50abe;
  }
  85% {
    text-shadow: 0px 0px 9px #0097e8, 0px 0px 15px #f50abe, 0px 0px 5px #0097e8;
  }
}

You can set as many percentages as you would like, depending on how many variations you have. Just make sure that the name of your @ keyframes rule matches the value you passed to the animation property.
